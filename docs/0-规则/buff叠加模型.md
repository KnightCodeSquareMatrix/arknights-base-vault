# Buff 叠加模型

## 一句话

干员技能不是同时生效的，有固定的先后次序。这个次序解释了为什么巫恋归零不影响裁缝、为什么孑在灵知体系里表现不同。

## 技能生效的先后次序

1. **全局资源先生成** — 感知信息（爱丽丝在宿舍产生梦境→转化为感知）、人间烟火（令/夕/重岳产生）等先进入全局池。这些资源由中枢和宿舍的干员先生产好，等着后续被贸易站或制造站的干员消费。

2. **固定加成写入** — 纯 +N% 效率和 +N 订单上限先写入：银灰 +20% 效率 +4 上限、可露希尔 +10% 效率、但书 +30% 效率、讯使 +15% 效率 +2 上限等。这些是基础数字，后续所有涉及效率的计算都以它们为准。

3. **中枢技能跨房间生效** — 控制中枢干员的技能跨房间影响到目标设施：灵知让每个谢拉格干员 −15% 效率 +6 上限、焰尾让每个红松骑士团干员 +10% 经验制造效率、八幡海铃让每个在贸易站的叙拉古干员 +5% 效率。这一步发生在固定加成之后，所以灵知的 −15% 扣在已经加了固定效率的数字上。

4. **订单上限汇总** — 把所有干员的 +上限（银灰 +4、讯使 +2、灵知 +6）和 −上限全部加起来，算出最终订单上限。孑「市井之道」对上限的缩减在这一步发生：每 10% 同房其他人效率缩减 1 个上限。他读的是固定加成和中枢技能之后的总效率。

5. **订单差值效率** — 上限算完后，基于「上限 − 当前订单数」的差值来计算额外效率。典型：孑「摊贩经济」每差 1 笔订单 +4% 效率。上限越高、差值越大、孑加的越多。

6. **基于他人效率再计算** — 基于同房**其他人**的效率来追加自身加成。典型：雪雉「天道酬勤」每 5% 他人效率为自己 +5%（最多 +35%）、诗怀雅「招商引资」每个订单上限提供效率。如果同房其他人效率很低，这一步可能不触发或收益很小。

7. **归零与吸收** — 巫恋「低语」将同房其他人的效率全部归零（不管之前累积了多少），每人为巫恋自身 +45%。佩佩也有类似机制。归零之后，下一步不受影响——因为那一步不依赖效率数字。

   另外，**清零优先规则**：当深海链（集群狩猎·β）与自动化组（自动化·α/β、仿生海龙、配合意识）同在一个制造站时，自动化组的清零效果优先生效 → 深海猎人加成被清零。两套体系不能共享同一制造站，必须分站部署。

8. **订单类型改变** — 改变订单的性质：裁缝 α/β 提高高赤金订单的出现率、但书让订单追加赤金数量、可露希尔把订单替换为特别订单、佩佩生成独立订单。这些技能的效果来自技能本身的描述，**不看干员的效率数字**，所以上一步的归零对它无效。

9. **心情消耗计算** — 最后计算每个干员的心情消耗速率。此时效率、上限、订单类型已全部确定。

## 几个重要推论

### 为什么巫恋归零不影响裁缝

巫恋的归零发生在先，裁缝改变订单类型发生在后——归零先执行。但裁缝的效果是「提高 4 赤金订单的出现率」，这个能力来自技能本身，不是来自干员自身的效率数字。归零只把效率数字清零，不影响裁缝技能发挥作用。

这就是巫恋核第三人必须是裁缝 β 的原因：裁缝 β 不需要自己有什么效率，只看技能有没有解锁、贸易站是不是 Lv.3、有没有足够长时间连续在岗。

### 为什么孑在灵知体系里表现不同

灵知的 −15% 效率在固定加成之后、孑市井之道之前生效。灵知先降低了效率，孑读到的是已经被灵知减过的值——所以孑对订单上限的缩减比没有灵知时更少。

灵知减效率 → 孑少压上限 → 银灰的 +4 上限有更大空间发挥。这就是灵孑银崖组合的基础。

### 为什么雪雉和孑同房可能不触发

雪雉「天道酬勤」需要读同房**其他人**的效率来叠自身加成，但它发生在归零之前。孑本身效率为 0，如果房里没有第三个干员提供效率，雪雉读不到东西就不生效。有第三方（如银灰 +20%）时雪雉才触发。

### 固定加成和后续加成的区别

固定加成（银灰 +20%、可露希尔 +10%）在前面，会被孑市井之道看到并用来计算上限缩减，也会被雪雉天道酬勤看到并叠加。

订单差值效率和雪雉那类加成在订单上限汇总之后才计算，不会被孑市井之道看到。这就是精 0 孑和精 1+ 孑差异巨大的原因：精 1 的市井之道多压了上限，又在后面少了摊贩经济的差值效率。没有灵知配合时，精 0 孑可能比精 1 更实用。

### 裁缝技能的效果是叠加的，不是覆盖的

如果多人同时有裁缝技能（如 α + β），两个都生效，最终订单分布是两个效果的叠加。但 α+β 同在一个房里的收益不如单上 β 再加其他干员——因为裁缝不直接提高订单效率，多占一个位浪费输出。

### 深海链和自动化组为什么不能同站

深海链的「集群狩猎·β」和自动化组的「自动化·α/β」「仿生海龙」都作用于制造站效率。当它们同在一个制造站时，清零效果优先生效——自动化组的清零规则先执行，直接将深海猎人的加成抹掉。这不是互斥锁，而是清零优先于正加成的叠加次序决定的。实践中两套体系必须分站部署。

## 对实际排班的影响

- 裁缝 β 干员必须精 2：精 1 只有裁缝 α，精 2 才能升级到裁缝 β。在巫恋核中必须精 2 才能发挥 85% 四赤金峰值
- 巫恋核的第三人只看裁缝档次（β > α）和心情消耗，不看干员自身的效率
- 灵知体系必须完整搭配（灵知中枢 + 孑精 1+ + 银灰 + 喀兰工具人），单独塞一个在不完整的体系里可能反而降效
- 赤金虚拟产线的干员（鸿雪、绮良等）放在巫恋核中不受归零影响，但不提供裁缝价值，不如直接上裁缝 β
